# Конвенции по работе с Supabase и UI-состояниями

Этот документ описывает строгие правила подключения UI-действий к Supabase во всех компонентах проекта:

1. **Единый клиент:** Все запросы к Supabase должны проходить через существующий клиент (обычно `src/lib/supabase.ts` или аналог). Запрещено создавать повторные инициализации клиента внутри компонентов.

2. **Состояния кнопок:** Состояние каждой кнопки-действия должно проходить жизненный цикл:
   - *Обычное (enabled)*
   - *Loading* (спиннер внутри кнопки, кнопка `disabled` на время запроса)
   - *Success* (короткий тост без слова "успешно" — просто констатация: "сохранено", "триггер создан", "прогон запущен")
   - *Error* (тост с конкретным текстом, что делать: "не удалось сохранить. проверьте соединение и попробуйте снова" — без префикса "Error:" и без первого лица).

3. **Сравнение состояний (Dirty check):** Формы с полями (настройки, триггеры, отчёты) — кнопка "сохранить" активна только при наличии несохранённых изменений. Это делается через сравнение текущего состояния формы с последним сохранённым снимком (используя `useState`/`useMemo`). Кнопка становится неактивной сразу после сохранения.

4. **Фоновые задачи (Background Jobs):** Действия с побочными эффектами дольше 2-3 секунд (запуск проверки по всем моделям, генерация плана оптимизации, экспорт отчёта) не должны блокировать интерфейс полноэкранным спиннером. Кнопка показывает "выполняется…", остальной интерфейс доступен. Результат обновляется через realtime-подписку Supabase (`supabase.channel(...).on('postgres_changes', ...)`) или через повторный fetch по завершении.

5. **Деструктивные действия:** Удаляющие действия (удалить проект, отключить интеграцию, удалить факт) требуют подтверждения через модалку. Для необратимых действий (удаление проекта) — дополнительно ввод названия проекта текстом для подтверждения.

6. **Внешние API (Edge Functions):** Действия, требующие внешнего API (Auto-PR, Google Docs export, OAuth-подключение интеграций), нужно оборачивать в Supabase Edge Function. Запрещено дёргать сторонние API напрямую из фронтенда, если требуются секретные ключи.
